# 自适应翻译 0.9.1

自适应翻译是一个浏览器英语阅读辅助插件。它不会把整篇网页直接替换成中文，而是在英文原文中标注生词，并在段落下方显示可联动的中文对照翻译。

## 核心功能

- 按英语水平标注生词，支持 L1-L5 阅读等级
- 段落下方显示中文对照翻译
- 鼠标悬停英文句子时，对应中文句子从灰色变黑变清晰
- 支持全面翻译模式和正文翻译模式
- 默认只翻译当前手动点击的页面；可手动开启连续阅读模式
- 点击“取消翻译”后恢复网页原文
- 支持 OpenAI、阿里云百炼/通义千问、火山方舟/豆包
- 默认关闭 reasoning/thinking，降低延迟和 token 消耗
- 支持查看最近 1 小时、24 小时、7 日、30 日的 token 用量统计
- 支持自定义提示词工作流
- 支持生词本、手动添加生词、筛选、排序和 CSV 导出
- Popup 支持快捷调整阅读等级、扫描模式、标注字号和标注颜色

## 适合谁

- 想读英文网页，但不想被整页机翻打断的人
- 希望逐步提升英文阅读能力的人
- 需要在技术文档、博客、新闻、论文页面中快速理解生词的人
- 希望使用自己的接口密钥和模型服务商的用户

## 安装方式

### Microsoft Edge 开发者模式安装

1. 下载或克隆本项目。
2. 打开 Edge，进入 `edge://extensions/`。
3. 打开右侧的“开发人员模式”。
4. 点击“加载解压缩的扩展”。
5. 选择本目录，也就是包含 `manifest.json` 的文件夹。
6. 安装完成后，浏览器右上角会出现 自适应翻译图标。

### Chrome 开发者模式安装

1. 打开 Chrome，进入 `chrome://extensions/`。
2. 打开“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择包含 `manifest.json` 的文件夹。

## 快速使用步骤

1. 解压插件文件夹，确认里面包含 `manifest.json`。
2. Edge 打开 `edge://extensions/`，Chrome 打开 `chrome://extensions/`。
3. 打开“开发者模式”。
4. 点击“加载解压缩的扩展”或“加载已解压的扩展程序”。
5. 选择 自适应翻译插件文件夹。
6. 点击浏览器右上角的 自适应翻译图标，进入设置页。
7. 选择服务商：OpenAI、阿里云百炼、火山方舟、智谱 AI、DeepSeek、Kimi 或 MiniMax。
8. 填写接口密钥、接口地址和模型名称。
9. 选择阅读等级，L1 标注最多，L5 标注最少。
10. 选择翻译范围：
   - 全面翻译模式：处理导航、按钮、目录、页脚和正文。
   - 正文翻译模式：主要处理文章正文。
11. 打开英文网页，点击“翻译当前页”。
12. 页面会在英文原文旁插入中文释义。
13. 点击“取消翻译”即可清除标注并恢复原文。

## 推荐模型

如果使用阿里云百炼，日常阅读建议优先使用 `qwen3.5-flash`。它速度快、成本低，适合网页生词标注这种短文本、多请求场景。

如果你更重视标注质量，可以尝试：

- `qwen-plus`
- OpenAI 的轻量模型
- 火山方舟中的豆包模型

不同服务商的模型名称可能会变化，请以各服务商控制台为准。

## 推理 / 思考模式

部分模型默认可能开启 reasoning 或 thinking。自适应翻译默认关闭这类模式，以减少等待时间和 token 消耗。

如果你觉得标注结果不够准确，可以在设置页打开“推理 / 思考模式”。打开后可能更准，但通常会更慢，也可能消耗更多 token。

## Token 用量统计

自适应翻译会根据模型接口返回的 `usage` 字段，在本地记录 token 用量。设置页的“用量统计”会显示：

- 最近 1 小时
- 最近 24 小时
- 最近 7 日
- 最近 30 日

统计会按服务商和模型分组展示。少数接口不返回 `usage` 时，自适应翻译只记录请求次数。

所有用量统计都保存在浏览器本地。你可以在设置页底部清空统计，清空前会出现二次确认。

## 隐私说明

自适应翻译不自带服务器。插件工作时，会把当前网页中需要标注的英文文本发送给你在设置页配置的模型服务商，例如 OpenAI、阿里云百炼或火山方舟。

自适应翻译会在浏览器本地保存以下配置：

- 接口密钥
- 接口地址
- 模型名称
- 阅读等级
- 提示词设置
- token 用量统计

请确认你信任所使用的模型服务商，并避免在敏感页面上使用插件。

## 权限说明

插件会请求以下浏览器权限：

- `storage`：保存接口配置、提示词、阅读等级和用量统计。
- `activeTab`：在你点击插件时读取并处理当前标签页。
- `<all_urls>` content script：让插件可以在网页中插入生词标注。

## 常见问题

### 为什么请求数很多？

自适应翻译会按页面文本块处理网页。长文章、导航、目录、页脚和按钮都可能产生独立请求。全面翻译模式覆盖更广，请求数会更多。

### 为什么 token 消耗比想象中高？

每次请求除了网页文本，还会带上提示词、阅读等级说明和返回格式要求。请求越多，重复发送的提示词越多，输入 token 就越高。

### 为什么有些地方没有标注？

可能原因包括：

- 文本不是英文或英文占比太低
- 当前阅读等级较高，模型判断无需标注
- 页面文本结构特殊，无法安全插入标注
- 模型返回为空或返回格式异常

可以尝试降低阅读等级，或切换到全面翻译模式。

### 切换页面后会自动翻译吗？

默认不会。自适应翻译默认只处理你手动点击“翻译当前页”的页面。

如果你在弹窗或设置页开启“连续阅读模式”，同一个标签页跳转到新页面后会继续标注。这个模式适合连续读英文资料，但会增加请求量和 token 消耗；余额不多时建议保持关闭。
